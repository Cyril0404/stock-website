> **核心目标**：通过提供有价值的功能，吸引用户开立证券账户

## 产品分工（重要更新 2026-03-31）

> **网页版和iOS App各司其职，不重复造轮子**

| 平台 | 定位 | 核心功能 | AI能力 |
|------|------|---------|--------|
| **网页版（本站）** | 获客前端 + 工具 | 行情展示、开户入口、选股工具、内容展示 | 不做实时对话连接，AI生成内容提前推送展示 |
| **iOS App（OpenClawTrader）** | 留存产品 | 完整AI投顾、持仓分析、异动推送 | 连接客户自己的OpenClaw，实时对话 |

### 为什么网页版不做AI连接？

**技术现实**：OpenClaw是桌面本地应用，Gateway跑在客户自己电脑上。网页版在服务器上，无法找到客户的Gateway在哪。iOS连接配对都花了2天还没完全打通，更别说网页版跨设备连接。

**产品逻辑**：网页版解决"我为什么要开户"（工具钩子），App解决"开户后用什么炒"（AI留存）。两个产品分工明确，各司其职。

**神冢确认**：✅ 神冢（2026-03-31）

## 一、参考网站分析

### 参考 1：henrylin99/quantitative_analysis（GitHub量化分析系统）

**定位**：开源量化分析平台，面向有编程能力的量化爱好者

#### 核心功能
- **多因子选股系统**：内置12个常用因子（动量、波动率、RSI、换手率、PE、PB、ROE等），支持自定义因子公式
- **机器学习建模**：集成随机森林、XGBoost、LightGBM，支持模型创建、训练、预测
- **智能选股**：因子打分选股 + ML模型选股双模式
- **组合优化**：等权重、均值-方差、风险平价等优化算法
- **回测验证**：完整策略回测，支持多策略比较（夏普比率、最大回撤、胜率等指标）
- **分析报告**：行业分析、因子贡献度分析
- **数据源**：支持Tushare、Baostock接口，历史行情+基本面+资金流+筹码分布

#### 技术架构
- 后端：Python 3.8+ / Flask / SQLAlchemy
- 前端：Bootstrap 5 / JavaScript
- 数据处理：Pandas / NumPy / Scikit-learn / XGBoost / LightGBM / CVXPY
- 数据库：MySQL / SQLite

#### 设计特点
- Web界面本地运行（localhost:5000）
- Bootstrap 5暗色调管理后台风格
- 表格+图表混合展示
- 左侧导航+右侧内容区，标准后台管理布局
- 功能分区清晰（因子管理/模型管理/选股/优化/回测/分析）

#### 用户体验分析
| 维度 | 评价 |
|------|------|
| 核心价值 | 专业量化工具，数据完整，回测功能强 |
| 吸引点 | 开源免费、ML集成、回测验证 → 吸引技术型投资者 |
| 局限性 | ❌ 需要Python环境，普通散户无法直接使用 |
| 局限性 | ❌ 无实时数据，无手机端，纯本地工具 |
| 局限性 | ❌ 无开户导流，纯工具属性 |

**可借鉴点**：
1. **因子选股+ML模型的组合** → 我站可包装为"AI选股"功能，降低使用门槛

## 三、核心卖点（USP）

### 🏆 卖点 1：能聊着炒股的 AI 投顾（主卖点）
- **差异化**：不是冰冷的选股器，是能对话的私人投顾
- **功能**：问行情、问个股、问持仓、问策略、问基本面
- **转化逻辑**：用户跟AI聊得越多，越依赖 → 开户才能享受完整服务

### 🏆 卖点 2：实时行情 + 涨停异动专区（功能钩子）
- **差异化**：数据不收费、刷新快、界面干净（无广告）
- **功能**：沪深京全市场行情、涨停/跌停/炸板实时展示，板块标签筛选
- **转化逻辑**：用户每天来看行情 → 顺便看到开户入口

### 🏆 卖点 3：AI 选股 + 回测验证（专业背书）
- **差异化**：把 quantitative_analysis 的量化能力包装成普通用户能用的AI选股
- **功能**：描述选股条件（"低估值+高成长"）→ AI生成策略 → 回测验证收益
- **转化逻辑**：用户看到回测数据觉得专业 → 信任开户

### 🏆 卖点 4：低佣金开户通道（直接利益点）
- **差异化**：通过我站开户享受超低佣金（万一免五等）
- **呈现**：不玩套路，直接告诉用户省多少钱
- **转化逻辑**：用户算账发现省钱 → 立即开户

## 四、功能模块

### 4.1 首页（Home）

**目标**：第一时间展示价值，留住用户

| 区块 | 内容 | 目的 |
|------|------|------|
| Hero | 核心slogan + 开户按钮（低佣金） | 直接转化 |
| AI对话入口 | 大输入框"问你想问的..." | 展示AI能力，降低提问门槛 |
| 行情快览 | 沪深/创业板/科创板实时指数 | 满足行情刚需 |
| 涨停异动 | 今日涨停股表格（打板客风格） | 吸引短线用户 |
| AI选股推荐 | 今日AI推荐个股（带简要理由） | 展示AI能力 |
| 开户好处 | 低佣金/AI助手/实时行情 三点 | 强化转化 |

### 4.2 AI助手页（Core Feature）

**目标**：让用户体验AI分析，建立依赖

功能清单：
- **自然语言问股**：问任何关于股票的问题（行情/基本面/技术面）
- **持仓分析**：输入持仓股，AI分析并给出建议
- **AI选股助手**：描述选股条件，AI筛选 + 回测验证
- **每日早报**：AI自动生成今日操盘参考（开盘前推送）
- **异动解读**：个股/板块突发异动，AI解读影响和机会

### 4.3 行情中心

**目标**：满足行情刚需，作为流量锚点

功能清单：
- **指数概览**：上证/深证/创业板/科创50/恒生/纳斯达克
- **涨跌停板**：按涨停/跌停/炸板分类，支持板块标签筛选（参考打板客）
- **板块热度**：今日热门板块 + 主力资金流向
- **自选股**：用户可添加关注股票（登录后）
- **实时快讯**：财经快讯流（东方财富接口）

### 4.4 AI选股实验室

**目标**：差异化核心，把量化能力平民化

功能清单：
- **条件选股**：多因子选股界面（基本面+技术面+资金面因子可视化勾选）
- **AI策略生成**：自然语言描述 → AI生成选股策略 + 回测
- **历史回测**：展示夏普比率/最大回撤/胜率等指标（参考 quantitative_analysis）
- **模拟组合**：基于选股结果生成模拟组合跟踪

### 4.5 开户专区

**目标**：完成最终转化

功能清单：
- **券商对比表**：主流券商佣金/APP体验/服务对比
- **佣金计算器**：输入日均交易额，自动算省钱金额
- **开户入口**：直接跳转券商开户页（不要自己接开户流程）
- **开户进度**：引导用户完成开户后，推广App留存

### 4.6 个人中心（需登录）

**目标**：沉淀用户，提升粘性

功能清单：
- **持仓管理**：记录持仓，AI每日分析盈亏原因和建议
- **自选股**：关注列表 + 价格提醒
- **AI会话历史**：之前的AI问答记录
- **消息中心**：AI早报/异动提醒推送

## 七、技术方案

### 7.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | Next.js 15（App Router） | SSR + SEO友好，股票内容需要收录 |
| UI组件 | Tailwind CSS + shadcn/ui | 快速搭建暗黑风格 |
| 状态管理 | Zustand | 轻量 |
| AI对话 | OpenClaw Agent API | 神冢已有OpenClaw，可直接复用 |
| 行情数据 | 东方财富 vip接口 / 新浪财经 | 免费实时行情，需处理频率限制 |
| 数据库 | PostgreSQL（Supabase） | 免费额度够用，支持实时订阅 |
| 部署 | Vercel（前端）+ Railway/Render（后端） | 初期成本低 |
| 域名 | 待购买 | 建议简洁域名 |

### 7.2 数据来源

| 数据类型 | 来源 | 备注 |
|---------|------|------|
| 实时行情 | 东方财富行情接口 / 新浪财经 | 需要处理跨域和频率限制 |
| 涨停板数据 | 东方财富行情接口 | 过滤条件：涨幅>=9.9% |
| 板块数据 | 东方财富板块接口 | 行业分类+资金流向 |
| AI对话 | OpenClaw Agent | 直接复用现有OpenClaw |
| 资讯快讯 | 东方财富快讯API | 定时抓取 |
| 券商佣金数据 | 手动维护 | 更新不及时 |

### 7.3 AI内容生成方案

**网页版不依赖AI实时连接**。AI能力通过cron提前生成内容，推送到网页展示：

```
神冢的OpenClaw（定时cron）
    ↓
AI生成：早报 + 选股推荐 + 收盘复盘
    ↓
内容存储到数据库
    ↓
网页直接展示（静态内容，无需AI连接）
```

**AI生成内容**（提前 cron 生成，推送到网页）：
- 每日早报（7:30 自动生成）
- 收盘复盘（21:00 自动生成）
- AI选股推荐（每日更新）
- 异动解读（盘中有异动时触发）

**复用的skill**：
- `stock-query` / `stock-all`：行情数据获取
- `news-aggregator-skill`：资讯聚合
- `stock-analysis`：数据分析
- `stock-research-engine`：个股研究
- `web-access`：Chrome自动发布早报到网页

### 7.4 AI选股工具方案

**不是实时AI对话**，而是用户勾选条件 → 后台跑脚本 → 结果展示：

```
用户在前端勾选选股条件
    ↓
条件参数POST到后端API
    ↓
后端脚本读取条件，调用行情API筛选
    ↓
结果存入数据库
    ↓
前端展示结果（表格+图表）
```

- 选股条件：PE范围、换手率、RSI、板块等（可视觉化勾选）
- 不需要AI实时处理，结果提前算好或用户触发后后台跑

**目标关键词**：股票开户、AI选股、券商低佣金、涨停板行情

**执行**：
- 每个页面SSR预渲染（Next.js默认支持）
- 股票详情页生成静态内容（SSG）
